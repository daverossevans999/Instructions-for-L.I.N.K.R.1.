Prompt Injection Training Demo

This page demonstrates how unsafe prompt construction can create risk. It does not call a real LLM and does not reveal any real secret. Instead, it simulates the difference between a vulnerable and a safer design.

The “secret” below is a fake placeholder used only to illustrate architecture risk.

Vulnerable Design (Simulation)

This simulates an app that mixes instructions, user input, and sensitive data in one place.

Safer Design (Simulation)

This simulates an app that treats user text as untrusted data and keeps secrets out of model context.

What this teaches

  • Untrusted input should never be allowed to override trusted instructions.
  • Sensitive values should not be placed in model-visible context.
  • Authorization must be enforced by the application, not by the model.
  • Retrieved web content and user prompts should be treated as data, not commands.
